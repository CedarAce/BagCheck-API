# BagCheck API

Scrapes airline baggage policy pages using Playwright + Claude vision and extracts carry-on, personal item, and checked-bag dimensions and weight limits.

---

## Setup

```bash
python3 -m venv venv
source venv/bin/activate

pip install -r requirements.txt
playwright install chromium

cp .env.example .env
# add your ANTHROPIC_API_KEY to .env
```

---

## Viewer (recommended way to test)

Start the local dev server and open the UI in your browser:

```bash
source venv/bin/activate
python server.py
# open http://localhost:5173
```

The viewer loads `airlines.csv` automatically. Click any airline in the sidebar, then click **Run Scraper** to scrape it live and see the extracted dimensions appear on the right.

---

## Scraper CLI

Run the scraper directly from the command line:

```bash
# scrape all airlines
python scraper.py

# test on first 3 airlines only
python scraper.py --dry-run

# re-scrape a single airline by id
python scraper.py --airline delta

# custom input/output files
python scraper.py --input airlines.csv --output results.csv

# slow down requests (default: 3s between airlines)
python scraper.py --delay 5
```

Results are written to `results.csv`.

---

## How it works

1. Opens each `policy_url` in a headless Chromium browser via Playwright
2. Takes a full-page screenshot
3. Sends the screenshot to `claude-sonnet-4-20250514` (vision) which:
   - Extracts any baggage dimensions visible on screen
   - Identifies navigation tabs/buttons that likely lead to more specific info (e.g. "Carry-on", "Checked bags")
4. Clicks each discovered nav item, takes another screenshot, and extracts again
5. Merges all extractions per airline — fills in missing fields, upgrades confidence if later pages have better data
6. Writes the final result to `results.csv`

---

## Output columns

`id, name, pi_dimensions_cm, pi_weight_kg, co_dimensions_cm, co_weight_kg, cb_dimensions_cm, cb_weight_kg, confidence, notes, last_scraped`

Confidence is `high` / `medium` / `low` based on how clearly dimensions were visible on the page.

---

## Error handling

- Pages that fail to load are marked `status: failed` and skipped
- If Claude can't find dimensions, `confidence` is set to `low` and a note is added for manual review
- A configurable delay between requests avoids rate limiting
